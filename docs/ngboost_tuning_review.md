# NGBoost 调参与评价复盘

> 2026-06-19；模型仍为 NGBoost，未更换模型。

## 调参记录的有效边界

Claude 会话中已运行 96 组二分类参数，覆盖 50–200 棵树、0.005–0.03 学习率、有/无样本权重和三种基学习器。该实验可以证明简单扩大参数搜索不能稳定解决问题，但不能将 0.587 当作最终泛化 AUC，原因是：

1. 验证段没有正类，96 组的 `auc_val` 全部为 `NaN`。
2. 脚本对每组参数查看了测试集，并按 `auc_test` 排序。
3. 因此 0.587 是多次查看同一测试集后的探索性上界，不是独立测试结果。

不再重复这 96 组网格。以后调参必须由有正/负类的验证段选参，测试段仅在方案锁定后评价一次。

## 后续准则

- 保留 GWT，不为提高指标人为删除主导因子。
- LogisticRegression 仅作为诊断基线，不视为已批准的模型替换。
- 环境变化量、位移速率/加速度已进入 SHAP；下一步应评价预警转折或 3/7 日提前量，而不是继续提高当日状态 AUC。
- 任何正式模型更换均需用户批准。
