# NGBoost 调参与评价复盘

> 复盘日期：2026-06-19。NGBoost 仍为当前预警概率主模型，没有更换模型。

## 1. 旧参数搜索记录

既有实验搜索了 96 组二分类参数，涉及 50-200 棵树、0.005-0.03 学习率、样本权重和三种基学习器设置。该实验属于探索性调试，不能作为正式参数选择或最终泛化评价，原因如下：

1. 验证段没有正类，全部 `auc_val` 为 `NaN`，因此验证目标不可估计。
2. 每组参数都计算了同一测试段的 AUC，并按测试 AUC 排序。
3. 最优测试 AUC 0.587 是重复查看测试集后的结果，不是预先锁定方案的一次独立评价。
4. 该实验只能说明当前验证设计无法支持可靠选参，不能证明 NGBoost 已达到性能上限，也不能证明继续调参必然无效。

旧 96 组结果保留作审计记录，但不得作为论文主结果，也不再按相同流程重复。

## 3. 暂停继续调参的原因

- 当前主要瓶颈是预测终点和验证设计，不是尚未搜索足够多的超参数。
- 继续使用相同测试段选参会进一步产生测试集过拟合。
- 当日状态 AUC 已很高但仍不超过持续性基线，继续追求该 AUC 不回答期刊的核心问题。
- Orange/red 和多数时间折缺少正类，无法稳定评价高等级与跨时期表现。
- 新增 onset 审计仅发现 3 个具有有效前置窗口的独立事件；h=1/3/7 虽有 3/9/21 个正日期，但不是相互独立事件。

因此当前暂停的是无效调参流程，不是停止使用 NGBoost。

## 4. 重新调参的前置条件

只有同时满足以下条件后才重新调节 NGBoost：

1. 1/3/7 日 onset 标签和连续事件提取已经实现。
2. 当前审计定义已经固定为连续黄色及以上、不跨绿色日期合并、最短 1 个活动日；其他定义只作敏感性分析。
3. 需要补充独立事件，使内部选参和外部评价不由单个事件决定。
4. 构造同时含正事件和负样本的内部时间验证折；不足时不计算 AUC/PR-AUC。
5. 确定主要选择指标和误报约束，不根据测试结果临时改指标。
6. 明确外层测试折，调参过程中不查看其模型性能。

## 5. 下一轮调参协议

### 5.1 嵌套时间验证

- 外层：滚动时间测试折，用于估计泛化表现。
- 内层：仅在外层训练期内进行扩展窗口验证和参数选择。
- 每折重新计算动态 V0、等速段、标准化和特征变换。
- 同一日期的 8 个测点不得跨分区。

### 5.2 选择指标

- 主要样本级指标：未来 onset PR-AUC。
- 概率约束：Brier score 和校准曲线。
- 事件级约束：事件召回率、首次命中提前量、误报事件数和误报持续时间。
- ROC-AUC、F1 和 accuracy 作为次要指标，不单独决定参数。

### 5.3 搜索与停止规则

- 搜索空间、随机种子和最大试验数在运行前写入版本控制。
- 搜索对象限于 NGBoost 的树数量、学习率、基学习器复杂度、行/列采样和类别权重。
- 不根据外层测试表现追加参数组合。
- 如果内部时间折的最优参数不稳定，应优先报告稳定性问题，而不是继续扩大网格。
- 参数锁定后，每个外层测试折只评价一次。
- 所有试验均保存配置、验证指标、运行状态和 Git 提交号，不只保存最佳结果。

## 6. 模型与特征决策

- 保留 GWT、位移速率和位移加速度，不因单次排名或指标变化直接删除。
- 特征去留由训练期消融、跨折稳定性和物理合理性共同决定。
- SHAP 只解释模型贡献，不将排名作为因果证据。
- 不使用 LightGBM。
- 任何主模型更换或新增学习器对照都需先向用户说明并获得批准。
