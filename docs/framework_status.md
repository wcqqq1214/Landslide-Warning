# Framework 指标覆盖与模型结果

本文依据 `framework.md`、当前代码和 2026-06-18 的完整重跑结果整理。状态区分为“已覆盖”“部分覆盖”“未进入模型”和“未实现”，避免把仅生成的数据列误认为已经参与建模。

## 1. Framework 指标覆盖情况

| Framework 内容 | 状态 | 当前实现 | 主要缺口与下一步 |
| --- | --- | --- | --- |
| 库水位 | 已覆盖 | `RWL` 进入 SHAP、ConvLSTM 和 NGBoost | 用滚动时序验证检查其贡献是否跨时期稳定 |
| 库水位速率 | 部分覆盖 | `RWL_rate` 已生成，并进入 ConvLSTM 和 NGBoost | 尚未进入 SHAP 候选集，无法回答 framework 中“筛选合适库水位速率指标”的问题 |
| 降雨影响 | 部分覆盖 | SHAP 使用逐日降雨 5 天滞后；ConvLSTM、NGBoost 使用 7/15/30 天累计降雨 | SHAP 尚未同时比较逐日降雨和多窗口累计降雨，当前不能确定最合适的降雨窗口 |
| 位移速率 | 部分覆盖 | 8 个测点速率已生成；NGBoost 使用速率均值和最大值 | 未作为 SHAP 候选因子，也未作为 ConvLSTM 显式通道；应只使用历史滞后值，避免目标泄漏 |
| 位移加速度 | 部分覆盖 | 8 个测点加速度已生成；NGBoost 使用加速度均值和最大值 | 未进入 SHAP 和 ConvLSTM，且聚合均值/最大值会丢失测点差异 |
| 改进切线角 | 未进入模型 | `features.py` 已计算 8 个测点的 `*_alpha` | 没有进入 SHAP、ConvLSTM、NGBoost 或最终预警判据，目前只是数据列 |
| 多指标融合 | 部分覆盖 | ConvLSTM 融合位移网格、水位、水位速率和累计降雨；NGBoost 融合速率、加速度及环境驱动 | 各模型使用的指标集合不一致，尚无消融实验说明每类指标带来的增益 |
| SHAP 指标筛选 | 部分覆盖 | NGBoost 回归和二分类均已输出 SHAP 排序 | 候选集缺少水位速率、累计降雨、速率、加速度和切线角；分类测试段无正类，解释结果缺少未来预警事件验证 |
| ConvLSTM 预测区间 | 部分覆盖 | 输出 P10/P50/P90，统计覆盖率、宽度和分位数交叉 | 目标覆盖率为 80%，实测仅 72.6%；`CAL_FRAC=0.0` 导致 conformal 校准未启用 |
| 动态预警指标 | 部分覆盖 | 三测点独立动态 V0，按 V0/5V0/10V0 输出绿黄橙红规则 | 数据只有绿色和黄色，橙色、红色没有训练样本；V0 只按训练段离线计算，不是在线滚动更新 |
| NGBoost 分类 | 部分覆盖 | 当前训练为数据实际支持的绿/黄二分类，保留四级规则 | 测试段全为绿色，无法评价预警召回率；不能把当前准确率当作四级预警有效性证据 |
| 自适应在线预警 | 未实现 | 当前为离线脚本管线 | 缺少新数据接入、滚动阈值更新、模型漂移监测、连续预警合并和在线输出接口 |

## 2. 尚缺的评价指标

| 评价对象 | 已有指标 | 建议补充 | 原因 |
| --- | --- | --- | --- |
| 位移点预测 | RMSE、MAE、持久性基线、RMSE skill | R2、NSE、分测点滚动窗口误差、误差置信区间 | 判断拟合程度、时段稳定性和结果不确定性；位移增量接近零时不宜把 MAPE 作为主指标 |
| 预测区间 | P10-P90 覆盖率、平均宽度、分位数交叉 | 测试集 pinball loss、Winkler/interval score、分测点校准误差、覆盖率置信区间 | 覆盖率和宽度必须联合评价，防止单纯扩大区间换取高覆盖率 |
| 预警分类 | 准确率、混淆矩阵；SHAP 阶段保存 precision、recall、F1、AUC | balanced accuracy、macro-F1、PR-AUC、Brier score、log loss、概率校准曲线 | 类别严重不平衡时，准确率和 ROC-AUC 容易产生误导 |
| 预警业务效果 | 暂无 | 事件召回率、漏报事件数、误报持续天数、平均提前量、每次事件首报时间 | 预警系统最终评价的是事件是否提前报出，而不是逐日标签是否相等 |
| 泛化能力 | 单次前 80%/后 20% 时间切分 | 滚动起点评估、按预警事件分块验证、各折均值和标准差 | 当前最后 20% 没有黄色事件，单次切分不能验证真实预警能力 |

## 3. 当前模型结果

### ConvLSTM

8 个测点、287 个测试日的汇总结果如下：

| 指标 | ConvLSTM | 持久性基线 | 判断 |
| --- | ---: | ---: | --- |
| RMSE | 0.313 mm | 0.340 mm | 降低 8.0%，有增益但幅度有限 |
| MAE | 0.149 mm | 0.181 mm | 降低 17.3% |
| P10-P90 覆盖率 | 72.6% | - | 低于 80% 目标 |
| 平均区间宽度 | 0.378 mm | - | 需与覆盖率联合判断 |
| 分位数交叉 | 0 / 2296 | - | 输出顺序稳定 |

三个论文重点测点的结果：

| 测点 | RMSE | 基线 RMSE | RMSE skill | 区间覆盖率 |
| --- | ---: | ---: | ---: | ---: |
| MJ9 | 0.107 | 0.125 | 13.8% | 37.3% |
| MJ1 | 0.118 | 0.183 | 35.7% | 55.7% |
| MJ3 | 0.190 | 0.223 | 14.7% | 53.3% |

结论：点预测在所有测点上都优于持久性基线，MJ1 提升最明显；但总体提升仍较温和，且三个重点测点的区间覆盖率明显不足。当前 `CAL_FRAC=0.0`，所以代码中的 conformal 校准没有实际生效，`qhat=0`。

### NGBoost 预警分类

- 全样本标签：绿色 1196 天、黄色 235 天、橙色 0 天、红色 0 天。
- 测试集共 287 天，实际全部为绿色；预测正确绿色 257 天，误报黄色 30 天。
- 测试准确率为 89.5%，但黄色 precision、recall、F1 和 AUC 无法在该测试段有效计算。

结论：当前结果只能说明模型在稳定期多数时候输出绿色，不能证明它能识别未来预警事件，也不能证明四级分类有效。

### SHAP 与 NGBoost 回归

- 位移增量回归 MSE 为 0.0277，对应 RMSE 为 0.167 mm/day。
- 回归 SHAP 前五项为 `disp_lag1`、`disp_lag3`、`GWT_lag1`、`disp_lag5`、`RWL_lag1`。
- 分类 SHAP 前五项为 `disp_lag1`、`GWT_lag1`、`disp_lag2`、`disp_lag4`、`disp_lag5`。
- 分类训练段有 301 个黄色样本行，测试段为 0，因此分类 SHAP 只能解释训练得到的决策规律，尚不能证明这些规律在未来黄色事件上成立。

当前解释显示近期位移、地下水位和库水位较重要，但 framework 特别关注的库水位速率、多窗口降雨、加速度和切线角还没有进入同一套 SHAP 比较，暂不能形成完整的指标筛选结论。

## 4. 改进优先级

| 优先级 | 改进项 | 预期作用 | 是否更换模型 |
| --- | --- | --- | --- |
| P0 | 将 ConvLSTM 训练段再划出校准集，启用现有 conformal 校准，并按测点报告覆盖率 | 先把 P10-P90 区间校准到接近 80% 目标 | 否 |
| P0 | 改为滚动起点/事件分块时间验证，保证至少部分验证折含黄色事件 | 获得可信的预警召回率、误报率和提前量 | 否 |
| P1 | SHAP 候选集加入 `RWL_rate`、累计降雨、历史速率、历史加速度和切线角 | 完整回答 framework 的指标筛选问题 | 否 |
| P1 | 对重点测点采用损失权重或测点独立输出头，并比较 2 天/7 天多尺度窗口 | 改善 MJ9、MJ1、MJ3 的点预测和区间覆盖 | 否，仍为 ConvLSTM |
| P1 | NGBoost 保存完整概率评价 CSV，增加 PR-AUC、Brier、校准曲线和事件级指标 | 避免只看准确率造成误判 | 否 |
| P2 | 做指标消融实验：位移、库水位、降雨、地下水、变形指标逐组加入 | 量化每类指标的真实增益 | 否 |
| P2 | 实现滚动 V0、数据漂移检查和连续预警事件合并 | 从离线实验推进到自适应在线预警 | 否 |

在完成 P0 和 P1 前，不建议更换模型。若后续确实需要比较其他模型，应先单独提出方案并取得批准。
