# 基于机器学习与多指标融合的滑坡位移预测及预警研究框架

> 文档性质：研究与验证协议。本文档区分“已实现”“探索性结果”和“待验证方案”，用于约束后续实验，避免根据测试结果反向修改研究问题或评价指标。

## 1. 研究目标与边界

### 1.1 研究对象

- 研究区：三峡库区藕塘滑坡。
- 数据频率：日尺度。
- 监测期：2016-07-01 至 2020-06-30，共 1461 日。
- 位移测点：MJ9、MJ1、MJ3、ATU1、ATU2、ATU3、ATU4、ATU5。
- 环境变量：降雨量、库水位、地下水位、温度、露点和相对湿度。

### 1.2 研究问题

1. 多源监测变量能否提高未来位移预测精度，并给出具有合理覆盖率的预测区间？
2. 位移速率、位移加速度、库水位变化率和多时间尺度降雨等候选指标，能否为未来预警状态提供超越持续性规则的增量信息？
3. 动态 V0、改进切线角和 NGBoost 概率能否形成可审计的多指标预警体系？
4. 模型在不同时间阶段、测点和预警事件上的表现是否稳定？

### 1.3 研究边界

- 当前结果属于单滑坡、内部时间验证，不代表跨滑坡泛化能力。
- 当前数据中的橙色和红色状态高度集中，不能据此宣称已验证高等级预警能力。
- SHAP 解释模型如何使用特征，不用于证明降雨、库水位或地下水位与位移之间的因果关系。
- 未经用户批准，不更换 ConvLSTM 或 NGBoost 主模型；新增模型即使仅作为对照，也需先说明目的并获得批准。

## 2. 任务定义

### 2.1 位移预测任务

对测点 `i` 和预测时点 `t`，使用不晚于 `t` 的信息预测 `t+h` 的累计位移：

```text
X(i, <=t) -> D(i, t+h), h in {1, 3, 7} days
```

- 已实现：`h=1` 日，输入窗口为 7 日，输出 P10、P50、P90。
- 待验证：`h=3` 和 `h=7` 日。不同预测步长应分别训练和评价，不允许以未来观测更新中间输入。
- 主要点预测：P50。
- 区间目标：名义 80% 的 P10-P90 预测区间。

### 2.2 当前状态分类任务

动态 V0 四级状态由测点 30 日位移变化量计算：

```text
V(i,t) = D(i,t) - D(i,t-30)
green:  V < V0
yellow: V0 <= V < 5V0
orange: 5V0 <= V < 10V0
red:    V >= 10V0
```

该任务用于描述和复核当日状态，是探索性辅助任务，不作为“提前预警”主要结论。由于相邻两日的 30 日窗口共享 29 日信息，必须与昨日状态持续性基线比较。

### 2.3 未来预警任务

未来预警是期刊论文的主要分类任务。对尚未处于黄色及以上状态的时点 `t`，定义：

```text
Y_onset(t,h) = 1, if any site-level warning in [t+1, t+h] >= yellow
Y_onset(t,h) = 0, otherwise
```

- 主要预测窗口：`h=3` 日和 `h=7` 日。
- `h=1` 日作为短临对照。
- 已经处于黄色及以上的日期不计作新的 onset 样本，持续状态另行评价。
- 当前审计定义：连续黄色及以上日期视为同一事件，默认不跨绿色日期合并，最短活动日为 1 日。
- 待敏感性分析：允许 1/3 日短间隔合并，并提高最短持续天数；不得根据外层测试表现选择定义。

如果有效预警事件数量不足以支持稳定估计，应报告事件数和不确定性，不以样本级高 AUC 替代事件级证据。

## 3. 候选指标及选择依据

| 指标组 | 指标 | 选择理由 | 研究定位 |
| --- | --- | --- | --- |
| 变形状态 | 累计位移、位移速率、位移加速度 | 描述变形水平、发展速度和转折 | 核心运动学指标 |
| 库水作用 | 库水位、库水位变化率 | 描述水位状态和涨落过程 | 外部环境指标 |
| 降雨作用 | 日降雨、7/15/30 日累计降雨 | 表征即时和滞后入渗影响 | 多时间尺度环境指标 |
| 地下水响应 | 地下水位及变化率 | 表征坡体水文响应 | 环境响应指标 |
| 气象背景 | 温度、露点、相对湿度 | 控制潜在季节和气象混杂 | 候选控制变量 |
| 规则指标 | 动态 V0 | 适应测点本底变形差异 | 主判规则 |
| 规则指标 | 改进切线角 | 衡量当前速率相对于等速阶段的加速程度 | 升级复核规则 |
| 不确定性 | P10/P50/P90 | 同时表达点预测和预测区间 | 风险信息 |

指标进入候选集应由监测机理、既有研究和可获得性共同决定；SHAP只用于解释与筛选，不作为候选指标合理性的唯一依据。所有变量必须记录单位、时间窗口、滞后方式、缺失处理和计算数据范围。

## 4. 阈值与规则的来源

### 4.1 动态 V0

当前实现仅使用前 80% 训练期，按测点计算 30 日位移速率。剔除高于训练期 90% 分位的加速月后：

```text
V0 = 1.5 * mean(V_steady) + 2 * std(V_steady)
```

当前 `30 日`、`90% 分位`、`1.5`、`2`、`5V0` 和 `10V0` 均应在论文中分别标明为文献规则、数据估计或工程设定。未找到直接文献依据的参数不得表述为普适阈值。

### 4.2 改进切线角

```text
alpha(t) = arctan(v(t) / v_eq)
```

- 文献原式先用等速阶段速率将累计位移坐标变换为时间量纲，再计算 `alpha = arctan(delta T / delta t)`；对当前日等间隔累计位移，代码中的速率比形式与其等价。
- 原文阶段判据：`alpha < 45` 为初始变形，`alpha` 约等于 45 为等速变形，`alpha > 45` 为加速变形；加速阶段再以严格 `> 80` 和 `> 85` 划分中加速与加加速（临滑）阶段。
- 当前四级映射：`alpha <= 45`、`45 < alpha <= 80`、`80 < alpha <= 85`、`alpha > 85`。颜色等级是本项目的工程映射，不是原文提出的颜色预警体系。
- `v_eq`：原文要求结合累计位移曲线和宏观变形迹象识别等速阶段后取平均速率；当前代码提供自动候选（15/30/60 日窗口）和人工阶段配置接口（`config/tangent_reference_stages.csv`），尚未进行专家确认。
- `v(t)`：原文使用等间隔监测增量；当前用于融合的日等级使用仅含历史数据的 3 日尾随线性斜率，以减弱日尺度噪声。
- 持续性：5 日窗口内至少 3 日达到相应等级，是本项目的工程确认规则，并非许强等（2009）的原始判据。

特征表同时保留原始角阶段等级、平滑角逐日等级和持续性工程等级。切线角在当前体系中不直接作为 NGBoost 输入，而是在融合层执行只升级不降级的复核；V0 与融合逻辑也不属于该切线角文献。专家等速阶段复核前，自动参数结果只能视为探索性。

### 4.3 敏感性分析

最终测试前固定以下敏感性组合：

- V0 月窗口：15、30、60 日。
- 稳定期截断分位数：0.85、0.90、0.95。
- 切线角等速候选窗口：15、30、60 日。
- 平滑窗口：1、3、5 日。
- 持续性规则：3 日/2 次、5 日/3 次、7 日/4 次。

敏感性分析用于判断结论是否依赖单一参数，不用于在测试集上选择最优组合。

当前回顾性敏感性分析已经按上述组合实现。V0 的等级序列相对默认组合一致率为 90.1%-100%，预警事件数为 2-6，说明参数会实质影响状态与事件定义。切线角在固定候选窗口内改变平滑和持续性规则时，一致率最低为 96.9%；但 15/60 日候选窗口相对 30 日默认组合的一致率仅约 29.8%-30.5%，主要原因是自动算法为 MJ1/MJ3 选择了数量级不同的 `v_eq`。这些结果不支持从三种窗口中反向选择“表现最好”的窗口；它们支持将专家等速阶段复核列为切线角确认性评价前的必要步骤。

## 5. 数据划分与防泄漏规则

### 5.1 当前结果的性质

现有前 80% 训练、后 20% 测试的结果已被用于多轮模型检查和参数比较，因此属于探索性时间留出结果，不再视为完全未触碰的确认性测试集。

### 5.2 后续内部验证

- 按时间顺序使用扩展窗口或滚动起点验证。
- 每一折的阈值、标准化参数、特征选择、等速段和模型参数只能由该折训练期确定。
- 验证期必须晚于训练期，禁止随机打乱后分层切分。
- 8 个测点在同一日期共享环境变量，数据划分必须按日期整体进行，不能将同一天拆入训练和验证。
- 调参使用内部时间验证；外层测试折只用于一次评价。
- 若获得新监测时段或新滑坡数据，应作为外部时间或空间验证，不参与任何参数选择。

## 6. 基线与比较原则

| 任务 | 必须报告的基线 |
| --- | --- |
| 位移预测 | 最后观测值持久性预测 |
| 当前状态分类 | 昨日状态持续性规则 |
| 未来 onset 预测 | “未来维持当前状态”规则和动态 V0 规则 |
| 概率预测 | 训练期事件率常数概率 |

主模型只有在预先规定的主要指标上超过对应基线，才能声称具有增量预测价值。模型复杂度本身不是有效性的证据。

## 7. 评价指标

### 7.1 位移预测

| 类别 | 指标 | 用途 |
| --- | --- | --- |
| 主要点预测指标 | MAE | 对极端误差较稳健 |
| 次要点预测指标 | RMSE | 强调较大误差 |
| 分位数指标 | P10/P50/P90 pinball loss | 评价各分位数质量 |
| 区间校准 | P10-P90 覆盖率 | 与名义 80% 比较 |
| 区间锐度 | 平均区间宽度 | 防止以过宽区间换取覆盖率 |
| 相对价值 | 相对持久性基线改进率 | 判断增量预测价值 |

### 7.2 预警分类

| 层级 | 指标 | 说明 |
| --- | --- | --- |
| 事件级主要指标 | 事件召回率、首次命中提前量 | 回答是否预警到、提前多久 |
| 事件级约束指标 | 误报事件数、误报持续时间 | 约束不可用的高频报警 |
| 样本级主要指标 | PR-AUC | 适用于正类稀少场景 |
| 概率质量 | Brier score、校准曲线、校准截距和斜率 | 评价概率是否可信 |
| 次要判别指标 | ROC-AUC | 便于与既有研究比较 |
| 阈值指标 | 召回率、精确率、F1 | 必须同时报告决策阈值 |
| 四分类指标 | 每类召回率、每类 F1、macro-F1、支持数 | 防止总体准确率掩盖稀有等级 |

当某一验证折只有一个类别时，不计算该折 AUC 或 PR-AUC，应报告“不可估计”和类别支持数，不以 0 或 1 代替。

## 8. 统计不确定性

- 点估计同时报告 95% 置信区间。
- 由于相邻日期相关且同日测点不独立，不能使用逐行独立 bootstrap。
- 样本级指标采用按日期连续块重采样；事件级指标采用事件重采样。
- 默认至少 1000 次重采样，并报告重采样单位、块长度和随机种子。
- 当事件数过少导致区间不稳定时，以事件计数、逐事件结果和定性限制为主，不进行过度统计推断。

## 9. NGBoost 调参与停止规则

- NGBoost 保持为预警概率主模型。
- 旧的 96 组搜索因验证段无正类且测试集参与排序，只能作为探索性实验。
- 当前 onset 盘点仅有 3 个可预测独立事件，不足以支持稳定的嵌套时间调参；在增加独立事件前暂停正式参数搜索。
- 新调参必须在含正负类的内部时间验证折上完成。
- 搜索空间在运行前锁定，至少记录 `n_estimators`、`learning_rate`、基学习器深度/叶节点、采样设置、类别权重和随机种子。
- 主要选择依据为未来 onset 任务的 PR-AUC、Brier 和事件级表现，而不是当日状态 AUC。
- 若 NGBoost 未超过持续性与规则基线，应如实报告，不通过扩大搜索空间反复使用外层测试集。

## 10. SHAP 分析规范

- SHAP 解释对象、背景数据、输出尺度和样本范围必须明确。
- 每个时间折独立训练和解释，比较重要性排序及方向是否稳定。
- 同时进行特征组消融：原始环境量、变化率、累计降雨、位移运动学量和切线角融合。
- 论文采用“模型贡献”“统计关联”“模型依赖”等表述，不采用“因果驱动”“导致”等因果措辞。
- 对高度相关的滞后特征，应说明贡献可能在相关变量之间分配，不能将单个 SHAP 排名解释为唯一物理机制。

## 11. 结果报告与复现

- 报告数据时间范围、样本数、事件数、各等级支持数和缺失情况。
- 报告所有时间折结果，不只报告最佳折或平均值。
- 图表中的指标必须能追溯到版本化 CSV；禁止手工修改图中数值。
- 保存依赖版本、随机种子、运行命令、模型参数和 Git 提交号。
- 结论必须区分位移预测、当前状态识别、未来 onset 预警和规则融合四个任务。
